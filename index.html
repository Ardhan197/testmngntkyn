Cakupan Full Sync Google Sheet

Auto-sync dan realtime pull mencakup: Karyawan, Absensi, Cuti, Shift, Assignment Shift, Resign, Blacklist, Master Cabang, Master Status, Pengaturan, Hak Akses/Login, Integrasi, Metadata Aplikasi, Maintenance, dan Drive Files.

Cloud-first aktif: perubahan dikirim otomatis ke Google Sheet, lalu data cloud ditarik berkala agar user/cabang lain melihat update terbaru.

Foto karyawan dan KTP akan otomatis dikirim ke Google Drive. Folder akan dibuat otomatis jika belum ada, lalu link Drive dikirim ke Google Sheet.

Ambil dari URL Google Sheet: docs.google.com/spreadsheets/d/SPREADSHEET_ID/edit. Link Apps Script bukan Spreadsheet ID.

Jika folder belum ada, Apps Script akan membuat folder ini otomatis di Google Drive, termasuk subfolder Foto Karyawan dan KTP.

Error SHEET_ID is not defined berarti kode Apps Script yang sedang ter-deploy belum punya variabel Spreadsheet ID. Salin ulang template Apps Script terbaru di bawah, isi ID Spreadsheet, lalu Deploy ulang Web App.

Restart & Kosongkan Data

Gunakan hanya saat maintenance besar. Fitur ini dipisah agar user bisa memilih reset lokal saja atau mengosongkan seluruh data aplikasi.

Zona Berisiko

Restart Ulang Lokal menghapus storage browser dan reload aplikasi. Integrasi cloud di browser ikut terhapus, tetapi Google Sheet tidak disentuh.

Kosongkan Seluruh Data mengosongkan data karyawan, absensi, cuti, shift, resign, blacklist, master, maintenance, dan pengaturan. Jika akses jaringan cloud aktif, hasil kosong akan ikut dikirim ke Google Sheet.
